Quoting CTS Texts in an HTML Document

In HTML, including images drawn from remote servers is simple: <img src="http://url-to-image"/>. If only it were so easy to draw passages of text from remote services! This page is a demonstration of how, with a little work behind the scenes, it is possible to include quotations almost as easily as we can include images.

This index.html page serves both as a guide and a template. At the top of this file, in the <head> element, are the links to the necessary Javascript and CSS files, and some variables that you can change to suit your needs.

This page will describe how to include CTS Texts, CITE Collection Objects, and CITE Image Objects.

The following guide assumes that you have some familiarity with the CTS/CITE Protocols.

URNs & HTTP-URIs

This kit allows you to work with either URNs or HTTP-URIs.

Assuming that a server is set up to accept HTTP-URIs for CTS Texts, CITE Objects, and CITE Images, it is simply a matter of inserting these URIs into <blockquote> elements as described below.

To use URNs, it is necessary to set values for the services that will handle CTS, CITE, and CITE-Image URNs. Set the urlOfCTS, urlOfCite, and urlOfImgService variables, in the <head> element of the HTML of this page.

(The current release assumes that all URNs of each type will be served from the same URL.)

Texts

Requirements: A known CTS Service and URNs or HTTP-URIs known to be available on that service.

Create a <blockquote>…</blockquote> element with a “class” value of “cts-text” and a “cite” attribute whose value is a CTS-URN or HTTP-URI. The element can contain some descriptive language. When the page loads, it will resolve to the text. (Note that HTML5 does not want <blockquote> elements to be empty, so throw some content inside the element; it will be replaced by the text pointed to by the citation, but will be visible in the source of the HTML document.)

So <blockquote class="cts-text" cite="urn:cts:greekLit:tlg0020.tlg001.fu01:1-10">Hesiod, Theogony 1-10</blockquote> becomes this:

Hesiod, Theogony 1-10 [If you are seeing this, the text has not loaded yet.]

You can include as many of these as you want:

Greek New Testament (Westcott & Hort), Matthew 4:1-4:3 [If you are seeing this, the text has not loaded yet.]

And here is one using an HTTP-URI: <blockquote class="cts-text" cite="http://folio.furman.edu/cts/urn:cts:greekLit:tlg0031.tlg001.fu001:1.1-1.10">New Testament, KJV, Matthew 1:1–1:10</blockquote>

New Testament, KJV, Matthew 1:1–1:10 [If you are seeing this, the text has not loaded yet.]

Objects

Requirements: A known CITE Collection Service and URNs or HTTP-URIs known to be available on that service.

Create a <blockquote>…</blockquote> element with a “class” value of “cite-collection” and a “cite” attribute whose value is a CITE-URN or HTTP-URI.

So <blockquote class="cite-collection" cite="urn:cite:botcar:sloane.414">Collection Object here</blockquote> becomes this:

Collection Object here [If you are seeing this, the object has not loaded yet.]

And another (this object has two fields that include Markdown formatting):

Collection Object here [If you are seeing this, the object has not loaded yet.]

Images

Requirements: A known CITE Image Service and URNs or HTTP-URIs known to be available on that service.

For this kit, we use the <img%gt; element, with a “class” attribute value of “cite-img”. The XSLT transforms that element into a code-block containing not only a thumbnail of the image, but a caption, information on rights, and links to a dynamic view.

  • Because the CITE Collection protocol allows Objects to have fields of type “markdown”, this kit can transform [Markdown](http://daringfireball.net/projects/markdown/) on the fly. Here’s some _Markdown_!
  • The scripts that resolve citations to texts will not work if the HTML files are loaded directly from the filesystem; they must be served over HTTP by Apache or some other web-server.
